Qyint Entscheidungsfindungsprozess

Notiz:

Einige Prozesse können sich in der Zukunft ändern. Lizensierung wird sich niemals verändern.

Fragen

Wie werden Fragen überhaupt dem Team vorgestellt?
Wenn ein Teammitglied den Administratoren eine Frage vorlegt, kann sie nicht abgelehnt werden.
Wenn ein Nutzer etwas ändern möchte, können sie die Änderung einem Teammitglied vorschlagen.
Teammitglieder können die Anfragen von normalen Nutzern ablehnen.

Wahlen

Wer darf Entscheidungen treffen?
Es gibt viele Arten von Wahlen.
Administratorwahlen (nur Admins wählen)
Teamwahlen (das gesamte Team wählt)
Administratorwahlen dauern eine Woche oder bis alle Administratoren gewählt haben.
Teamwahlen dauern zwei Wochen oder bis alle Teammitglieder gewählt haben.
Jeder kann eine Abstimmung, zum beispiel auf Discord, starten, um nach den Meinungen der Nutzer zu fragen.
Teammiglieder können selbst entscheiden, ob sie sich der Meinung der Nutzer beugen oder für sich selbst entscheiden.

§1 - Team

Wie wird das Team zusammengesetzt? ...
Administratoren können sich dazu entscheiden einen Nutzer in eine Team-Position zu befördern.
Solch eine Anfrage wird von den Administratoren abgestimmt.
Wenn 2/3 des Administratorteams sich dazu entscheiden, einen Nutzer in eine Position zu befördern, werden sie befördert.
Jeder Nutzer kann die Degradierung eines Teammitglieds vorschlagen, dafür wird auch von den Administratoren abgestimmt, wobei eine 2/3 Mehrheit entscheidet.
Solch eine Anfrage kann nicht abgelehnt werden.

Administratoren können sich dazu entscheiden, ein Teammitglied in eine Administrator-Position zu befördern.
Dafür wird vom ganzen Team abgestimmt.
100% des Teams müssen für eine Beförderung stimmen, um jemanden in die Position zu befördern.

§2 - Bestrafungen

Wie wird sich für eine Bestrafung entschieden? ...
Moderatoren und Teammitglieder in höheren Positionen entscheiden sich für eine Bestrafung.
Administratoren können sich dazu entscheiden eine Bestrafung zu verändern, zum Beispiel indem sie einen Ban zurückziehen. (Es ist die Pflicht der bestraften Nutzer einen Administrator bezüglich Beschwerden zu benachrichtigen.)
Sollten sich die Administratoren nicht für eine valide Strafe entscheiden können, wird vom ganzen Team abgestimmt.
Nur Administratoren dürfen Nutzerdaten (dh. Daten, die auf Qyint Servern gespeichert sind) löschen, nachdem dafür mit einer 2/3 Mehrheit abgestimmt wurde.

§3 - Service Verwaltung

Wer besitzt die Qyint Server? ...
Der Betreuer eines Services besitzt es zu 100%. (Außer sie teilen es beispielsweise mit einem anderen Betreuer.) Normalerweise betreuen die Administratoren alle Qyint Services.
Services, die nicht von Administratoren betreut werden, müßen als solches gekennzeichnet werden.
Notiz: Wir können niemals die Stabilität oder Sicherheit unserer Services garantieren, jedoch geben wir unser Bestes um gute Sicherheitspraktiken zu erhalten.
Wir werden niemals eure Daten mit Dritten teilen oder für Bösartiges nutzen.
Wie ihr das wissen wollt? Gar nicht. Wir könnten die Daten, die wir sammeln, was nur das ist, was ihr teilt, jeden Moment teilen, aber wir verprechen es euch das nicht zu tun.
Ehrlichkeit ist wichtig. Ihr habt keinen Grund uns zu vertrauen.

§4 - Repository Verwaltung

Wie wird sich für Änderungen entschieden?
Der Ersteller eines Repositories im Qyint-Community Github ist der Besitzer. (Außer sie übertragen beispielsweise den Besitz.)
Es gibt zwei Wege eine Änderung an einem Repository zu erlauben.
Option 1: Durch das Team
So wie andere Entscheidungen, wird auch eine Änderung, die von Teammitglidern vorgeschlagen wurde, entschieden.
Notiz: Wichtige Entscheidungen (wie bsw. Sicherheitsupdates) werden nur von den Administratoren entschieden.
Zum Schluß, entscheidet der Betreuer des Repositories, ob die Änderung aktepriert oder abgelehnt wird.
Option 2: Durch den Betreuer
Der Betreuer kann sich dazu entscheiden, eine Änderung zu akzeptieren.
Doch die Änderung kann retrospektiv vom Team abgelehnt werden.
Sollte ein Betreuer sich dazu entscheiden ein bösartiges Update zu veröffentlichen, oder sollte er nicht auf eine Mehrheitsabstimmung des Teams hören, können sie sanktioniert werden.
Sanktionen werden von den Administratoren entschieden.
Notiz: Tippfehler zu berichtigen benötigt keine autorisierung.
